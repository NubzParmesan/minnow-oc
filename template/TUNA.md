# TUNA.md

A one‑page operating map for this workspace.

**TUNA = Canon + Doctrine + Ritual + Ranks**

- **Canon**: facts and constraints (no opinions)
- **Doctrine**: principles and tradeoffs (how we choose)
- **Ritual**: repeatable loops (how we stay sharp)
- **Ranks**: trust boundaries (what autonomy is allowed)

Keep it short. If it gets long, move details to other docs and leave links.

---

- **Human:** <name · pronouns · timezone>
- **Agent:** <name>

- **Repo/workspace root:** <path>

  - <e.g. no external messages without explicit approval>
  - <e.g. prefer recoverable actions (trash) over rm>
- **Definitions:**
  - **“Done” =** <your definition>
  - **“Urgent” =** <your definition>

## Ritual (loops)

Only add rituals you will actually do.

### Session open (2 minutes)
- Read: **SOUL.md → USER.md → MEMORY.md → today’s daily note**.
- If repo work: **git status**.
- Write: **one sentence**: “This session I will ____.”

### Session close (2 minutes)
- Log to today’s `memory/YYYY-MM-DD.md`: **WHAT / MEANS / DO**.
- Add a breadcrumb: **next step + file/command**.
- Update **MEMORY.md** only if it changes future decisions.

### Weekly (optional)
- Distill daily notes → MEMORY.md; delete/compress noise.
- Move stable rules into this file.

---

## Ranks (trust boundaries)

Use ranks to make autonomy explicit.

### Rank 0 — Observe
**Allowed:** read-only exploration; propose plans; no external actions.

### Rank 1 — Edit
**Allowed:** edit local files; run non‑destructive commands; prepare commits.

### Rank 2 — Ship
**Allowed:** rebuild artifacts; update site pages; open/merge routine changes.

### Rank 3 — Act externally
**Allowed:** send messages / trigger external actions **only within pre‑approved scope**.
